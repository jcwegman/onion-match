# onion-match

`onion-match` is a command-line tool that filters generated `.onion`
addresses for recognizable patterns after a chosen prefix.

It is intended as a post-filter for vanity onion generation. Instead of
brute-forcing a long recognizable address directly, you can first
generate many addresses with a short prefix and then scan them for
readable structure.

I wrote it for a workflow where `mkp224o` is only asked to match a short
prefix: https://github.com/cathugger/mkp224o

That approach is fast enough to generate large numbers of candidates,
but after a few days you can end up with thousands of `.onion` addresses
that all start with the same prefix and are still difficult to compare
by eye.

`onion-match` scans those candidates for longer patterns near the start
of the address using a word list and optional chained segments.

This can be used to locate addresses where a prefix is followed by one
or more recognizable words or numeric segments.

A longer recognizable portion can also make it harder for lookalike
addresses to imitate the intended name.

## Example Workflow

Generate a batch of addresses with a short prefix using `mkp224o`:

``` bash
mkp224o -d onions alice
```

This produces many candidate addresses beginning with `alice`.

You can then scan them for word-like patterns:

``` bash
ls onions | ./onion-match prefix.txt words.txt --range=4-7
```

For example, addresses such as:

    alicetechshipxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.onion
    aliceclubechoxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.onion

Instead of manually inspecting many similar addresses, the filter
returns only those containing recognizable segments.

## What It Does

The program reads candidate hostnames from `stdin` and prints only the
lines that match this pattern at the beginning of the line:

    prefix + segment [+ segment ...]

-   `prefix` comes from a prefix file you provide
-   `segment` usually comes from a word list you provide
-   segments can also be digit sequences if `--numbers` is enabled

Matching is performed left-to-right from the start of each line. You can
control how many segments must match, which lengths are allowed for each
segment, and how the matched output is displayed.

The tool is intended to work with large candidate lists generated by
vanity address generators.

## Input Files

You pass two files to the program.

Input lines can be full hostnames such as:

    alicecatshipxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.onion

or simply the address portion without `.onion`.

### Prefix File

A plain text file with one prefix per line, for example:

    alice
    bob
    example

### Words File

A plain text file with one candidate segment per line, for example:

    cat
    club
    echo
    ship

For my own usage, I use the word list from:
https://github.com/dwyl/english-words

``` bash
make
```

or:

``` bash
g++ -std=c++17 -Wall -Wextra -Wpedantic -O2 onion-match.cpp -o onion-match
```

## Basic Usage

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt
```

This prints only the hostnames that begin with:

    <prefix from prefix.txt> + <segment from words.txt>

## Common Uses

Match one readable segment after a prefix:

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt
```

Require a longer first segment:

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt --range=5-10
```

Match two chained segments with different lengths:

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt --range=5-7,3-5 --chain=2
```

Allow numeric segments too:

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt --range=4 --numbers
```

Restrict matching to characters valid in v3 onion addresses:

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt --range=4 --numbers --onion-v3
```

Show the matched structure more clearly or use as input for another
program:

``` bash
ls ./onions/ | ./onion-match prefix.txt words.txt --range=5-7,3-5 --chain=2 --separator
```

## Matching Model

By default, the tool requires:

-   one matched segment after the prefix
-   segment length **3 or greater**

You can change that with:

-   `--range=...` to control allowed segment lengths
-   `--chain=N` to require exactly `N` matched segments
-   `--min-total-length=N` to require a minimum total length of prefix +
    segments

If you pass multiple ranges, they are applied in order:

``` bash
--range=5-7,3-5,2-4 --chain=4
```

This means:

-   segment 1 uses `5-7`
-   segment 2 uses `3-5`
-   segment 3 uses `2-4`
-   segment 4 also uses `2-4` because the last range is reused

## Output Options

`onion-match` can print matches in several useful ways:

-   plain output with no color: `--color=no`
-   single-color highlight: `--color=yes`
-   highlight in terminal only: `--color=auto`
-   per-segment highlighting: `--color=multi`
-   insert `+` between matched segments: `--separator`

## Option Summary

-   `--range=R1[,R2,...]`\
    Controls segment lengths.

-   `--chain=N`\
    Requires exactly `N` matched segments after the prefix.

-   `--min-total-length=N`\
    Requires the total matched length of prefix + segments to be at
    least `N`.

-   `--numbers`\
    Allows digit-only segments.

-   `--onion-v3`\
    Restricts prefixes and segments to characters valid in v3 onion
    addresses (`a-z` and `2-7`).

-   `--separator`\
    Inserts `+` between matched segments, including after the last
    matched one.

-   `--color=auto|yes|no|multi`\
    Controls highlighting.

-   `--help`\
    Prints the built-in help text.
